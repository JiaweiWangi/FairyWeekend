> 版本：v1.0.0

## 概述

「今日人设」是一个创意周末活动规划应用，用户抽取人设卡后，AI 会基于该人设生成个性化的城市剧情路线。

本文档描述项目中的所有 API 接口、数据模型和数据库结构。

## 后端 API (Supabase Edge Functions)

所有 Edge Functions 均部署在 Supabase，运行时为 Deno。

**通用请求头：**

```
Content-Type: application/json
Authorization: Bearer <SUPABASE_ANON_KEY>
```

**CORS 配置：**

```
Access-Control-Allow-Origin: *
Access-Control-Allow-Headers: authorization, x-client-info, apikey, content-type
Access-Control-Allow-Methods: POST, OPTIONS
```

### generate-quest

生成人设驱动的城市剧情路线。

**端点：** `POST /functions/v1/generate-quest`

#### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `card` | [PersonaCard](#personacard) | ✅ | 人设卡信息 |
| `city` | `string` | ❌ | 城市/区域名称，如 `"上海·静安区"` |
| `lat` | `number` | ❌ | 用户位置纬度 (WGS84) |
| `lng` | `number` | ❌ | 用户位置经度 (WGS84) |
| `time_period` | `string` | ❌ | 时间段，默认 `"下午"` |
| `companion` | `string` | ❌ | 同伴情况，默认 `"独行"` |

#### 请求示例

```json
{
  "card": {
    "id": "card_001",
    "rarity": "SR",
    "identity": "在异乡漂泊的植物学家",
    "mood": "对城市重新感到好奇",
    "mission": "寻找三种让你感到安心的气味",
    "colors": ["#f0e6d3", "#8fbc8f", "#d4a896"],
    "illustration_keyword": "botanical_wanderer"
  },
  "city": "上海·静安区",
  "lat": 31.2304,
  "lng": 121.4737,
  "time_period": "下午",
  "companion": "独行"
}
```

#### 响应参数

| 参数 | 类型 | 说明 |
|------|------|------|
| `journey` | [Journey](#journey) | 生成的剧情路线 |
| `city` | `string` | 解析后的城市名称 |
| `poi_count` | `number` | 候选 POI 数量 |

#### 响应示例

```json
{
  "journey": {
    "story_opening": "你推开那扇吱呀作响的木门，一股熟悉的青草味扑面而来...",
    "emotion_arc": {
      "start": "好奇",
      "end": "安心"
    },
    "scenes": [
      {
        "order": 1,
        "scene_name": "有阳光漏进来的角落",
        "location_name": "静安公园",
        "location_type": "公园",
        "location_hint": "上海市静安区南京西路1649号",
        "persona_narrative": "作为一名植物学家，你习惯性地扫视...",
        "action_task": "找到一棵叶子形状特别的树，拍下它的特写",
        "stay_minutes": 30,
        "emotion_tags": ["宁静", "好奇", "自然"],
        "meituan_keyword": "公园"
      },
      {
        "order": 2,
        "scene_name": "飘着咖啡香的街角",
        "location_name": "某独立咖啡馆",
        "location_type": "咖啡",
        "location_hint": "愚园路附近",
        "persona_narrative": "咖啡豆的香气让你想起...",
        "action_task": "点一杯手冲，问店员豆子的产地故事",
        "stay_minutes": 45,
        "emotion_tags": ["温暖", "专注"],
        "meituan_keyword": "独立咖啡馆"
      }
    ],
    "closing": "傍晚的阳光把影子拉得很长，你把这些气味都记在了心里..."
  },
  "city": "上海·静安区",
  "poi_count": 15
}
```

#### 错误响应

| 状态码 | 错误码 | 说明 |
|--------|--------|------|
| 400 | `missing card` | 缺少必要的人设卡信息 |
| 500 | `LOVABLE_API_KEY not configured` | AI 服务未配置 |
| 429/402 | `ai_error` | AI 服务调用失败（限流或余额不足） |
| 500 | `parse_error` | AI 返回内容解析失败 |

#### 内部流程

1. **坐标转换**：如果提供 `lat/lng`，将 WGS84 坐标转换为 GCJ02（高德坐标系）
2. **逆地理编码**：如果未提供 `city`，使用高德 API 反向解析城市名称
3. **POI 搜索**：根据人设身份和状态，搜索相关类型的热门地点
4. **AI 生成**：调用 Lovable AI Gateway (Gemini 2.5 Flash)，生成结构化路线

### record-quest

保存已完成的副本记录，并更新玩家长期画像。

**端点：** `POST /functions/v1/record-quest`

#### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `player_key` | `string` | ❌ | 玩家唯一标识，默认 `"default"` |
| `character_class` | `string` | ✅ | 角色职业（人设身份） |
| `emotion` | `string` | ❌ | 情绪状态 |
| `city` | `string` | ❌ | 城市名称 |
| `quest` | `Quest` | ✅ | 副本详情 |
| `stages_unlocked` | `number` | ❌ | 已解锁关卡数，默认 `0` |
| `rating` | `number` | ❌ | 用户评分 (1-5) |
| `feedback` | `string` | ❌ | 用户反馈文本 |
| `liked_stage_orders` | `number[]` | ❌ | 用户点赞的关卡序号数组 |

#### Quest 结构

```typescript
interface Quest {
  quest_name: string;        // 副本名称
  quest_brief?: string;      // 副本简介
  stages: Array<{
    order: number;           // 关卡序号
    stage_name: string;       // 关卡名称
    location_name: string;   // 地点名称
    location_type?: string;  // 地点类型
    emotion_tags?: string[]; // 情绪标签
  }>;
}
```

#### 请求示例

```json
{
  "player_key": "user_abc123",
  "character_class": "在异乡漂泊的植物学家",
  "emotion": "对城市重新感到好奇",
  "city": "上海·静安区",
  "quest": {
    "quest_name": "气味收集记",
    "quest_brief": "在静安区寻找三种让人安心的气味",
    "stages": [
      {
        "order": 1,
        "stage_name": "有阳光漏进来的角落",
        "location_name": "静安公园",
        "location_type": "公园",
        "emotion_tags": ["宁静", "好奇", "自然"]
      },
      {
        "order": 2,
        "stage_name": "飘着咖啡香的街角",
        "location_name": "某独立咖啡馆",
        "location_type": "咖啡",
        "emotion_tags": ["温暖", "专注"]
      }
    ]
  },
  "stages_unlocked": 2,
  "rating": 5,
  "feedback": "很喜欢这次路线，咖啡店推荐得很准！",
  "liked_stage_orders": [1, 2]
}
```

#### 响应参数

| 参数 | 类型 | 说明 |
|------|------|------|
| `ok` | `boolean` | 操作是否成功 |
| `profile` | `string` | 更新后的玩家画像描述 |
| `loved` | `string[]` | 更新后玩家喜欢的标签列表 |

#### 响应示例

```json
{
  "ok": true,
  "profile": "偏爱安静自然的场所，喜欢独立咖啡馆和小众公园，节奏偏慢，偏好独处时光",
  "loved": ["安静", "自然", "独立咖啡", "公园", "手冲"]
}
```

#### 错误响应

| 状态码 | 错误码 | 说明 |
|--------|--------|------|
| 400 | `missing fields` | 缺少必要字段 |
| 500 | `supabase env missing` | 数据库环境变量未配置 |

#### 内部流程

1. **插入历史记录**：将副本数据写入 `quest_history` 表
2. **读取旧画像**：从 `dm_memory` 表获取玩家历史画像
3. **合并访问地点**：将新访问的地点合并到 `visited_pois`
4. **AI 更新画像**：调用 AI 分析用户偏好变化，生成新画像
5. **更新画像表**：Upsert 到 `dm_memory` 表

### resolve-location

将 WGS84 经纬度反向解析为中文城市区域名称。

**端点：** `POST /functions/v1/resolve-location`

#### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `lat` | `number` | ✅ | 纬度 (WGS84) |
| `lng` | `number` | ✅ | 经度 (WGS84) |

#### 请求示例

```json
{
  "lat": 31.2304,
  "lng": 121.4737
}
```

#### 响应参数

| 参数 | 类型 | 说明 |
|------|------|------|
| `label` | `string` | 格式化的城市区域标签，如 `"上海·静安区"` |
| `city` | `string` | 城市名称 |
| `district` | `string` | 区县名称 |
| `province` | `string` | 省份名称 |
| `formatted` | `string` | 完整格式化地址 |
| `gcj` | `object` | 转换后的 GCJ02 坐标 |

#### 响应示例

```json
{
  "label": "上海·静安区",
  "city": "上海",
  "district": "静安区",
  "province": "上海",
  "formatted": "上海市静安区南京西路附近",
  "gcj": {
    "lng": 121.4802,
    "lat": 31.2361
  }
}
```

#### 错误响应

| 状态码 | 错误码 | 说明 |
|--------|--------|------|
| 400 | `lat/lng required` | 缺少必要参数 |
| 500 | `AMAP_WEB_API_KEY not configured` | 地图服务未配置 |
| 502 | `regeo failed` | 地理编码失败 |

## 数据模型

### PersonaCard

人设卡定义。

```typescript
interface PersonaCard {
  id: string;                    // 卡片唯一标识，如 "card_001"
  rarity: Rarity;                // 稀有度：N | R | SR | SSR
  identity: string;              // 身份描述
  mood: string;                  // 今日状态
  mission: string;               // 今日使命
  colors: string[];              // 卡片配色（3色十六进制）
  illustration_keyword: string;  // 插画关键词
  cover?: string;                // 卡面插画 URL
}

type Rarity = "N" | "R" | "SR" | "SSR";
```

**稀有度权重：**

| 稀有度 | 全称 | 权重 |
|--------|------|------|
| N | Normal | 50% |
| R | Rare | 30% |
| SR | Super Rare | 15% |
| SSR | Super Super Rare | 5% |

---

### Journey

完整的剧情路线。

```typescript
interface Journey {
  story_opening: string;         // 故事开篇（30-50字）
  emotion_arc: {                 // 情绪弧线
    start: string;              // 起始情绪
    end: string;                // 结束情绪
  };
  scenes: JourneyScene[];        // 场景列表（3-4个）
  closing: string;              // 今日结语（60-100字）
}
```

---

### JourneyScene

单个场景节点。

```typescript
interface JourneyScene {
  order: number;                // 场景序号
  scene_name: string;           // 诗意命名（6-10字）
  location_name: string;        // 地点名称
  location_type: string;        // 地点类型
  location_hint: string;        // 地址提示
  persona_narrative: string;    // 人设视角叙事
  action_task: string;          // 具体行动任务
  stay_minutes: number;         // 建议停留时长（分钟）
  emotion_tags: string[];       // 情绪标签
  meituan_keyword: string;      // 美团搜索关键词
}
```

### JourneyRunState

运行时状态（存储在 sessionStorage）。

```typescript
interface JourneyRunState {
  card: PersonaCard;            // 当前人设卡
  city?: string;                // 当前城市
  journey: Journey;             // 当前路线
  completedSceneOrders: number[];  // 已完成场景序号
  sceneRecords?: Record<number, SceneRecord>;  // 场景打卡记录
  createdAt: number;            // 创建时间戳
}
```

### SceneRecord

场景打卡记录。

```typescript
interface SceneRecord {
```

## 数据库结构

### quest_history

副本历史记录表。

| 列名 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | `UUID` | PK, DEFAULT `gen_random_uuid()` | 主键 |
| `player_key` | `TEXT` | NOT NULL, DEFAULT `'default'` | 玩家标识 |
| `character_class` | `TEXT` | NOT NULL | 角色职业 |
| `emotion` | `TEXT` | - | 情绪状态 |
| `city` | `TEXT` | - | 城市名称 |
| `quest` | `JSONB` | NOT NULL | 副本详情 |
| `stages_unlocked` | `INT` | NOT NULL, DEFAULT `0` | 已解锁关卡数 |
| `rating` | `INT` | - | 用户评分 |
| `feedback` | `TEXT` | - | 用户反馈 |
| `liked_stage_orders` | `INT[]` | DEFAULT `'{}'` | 点赞关卡序号 |
| `created_at` | `TIMESTAMPTZ` | NOT NULL, DEFAULT `now()` | 创建时间 |

**索引：** `idx_quest_history_player (player_key, created_at DESC)`

---

### dm_memory

玩家长期画像表。

| 列名 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `player_key` | `TEXT` | PK | 玩家标识（主键） |
| `profile` | `TEXT` | NOT NULL, DEFAULT `''` | 画像描述 |
| `loved_tags` | `TEXT[]` | DEFAULT `'{}'` | 喜欢的标签（最多8个） |
| `disliked_tags` | `TEXT[]` | DEFAULT `'{}'` | 不喜欢的标签（最多5个） |
| `visited_pois` | `TEXT[]` | DEFAULT `'{}'` | 访问过的地点（最多60个） |
| `total_runs` | `INT` | NOT NULL, DEFAULT `0` | 累计场次 |
| `updated_at` | `TIMESTAMPTZ` | NOT NULL, DEFAULT `now()` | 更新时间 |

**安全策略：** 两张表均启用 Row Level Security，无公开访问策略。Edge Functions 通过 Service Role Key 绕过 RLS。

## 错误处理

所有 API 返回的错误响应遵循统一格式：

```json
{
  "error": "<error_code>",
  "status": <http_status>,
  "detail": "<error_detail>"
}
```

**常见错误码：**

| 错误码 | HTTP 状态码 | 说明 |
|--------|--------------|------|
| `missing card` | 400 | 缺少人设卡信息 |
| `missing fields` | 400 | 缺少必要字段 |
| `lat/lng required` | 400 | 缺少经纬度参数 |
| `LOVABLE_API_KEY not configured` | 500 | AI 服务密钥未配置 |
| `AMAP_WEB_API_KEY not configured` | 500 | 地图服务密钥未配置 |
| `supabase env missing` | 500 | 数据库环境变量未配置 |
| `ai_error` | 429/402 | AI 服务调用失败 |
| `parse_error` | 500 | AI 返回内容解析失败 |
| `regeo failed` | 502 | 地理编码失败 |

## 调用示例

```typescript
import { createClient } from '@supabase/supabase-js';

const supabase = createClient(
  import.meta.env.VITE_SUPABASE_URL,
  import.meta.env.VITE_SUPABASE_PUBLISHABLE_KEY
);

const { data, error } = await supabase.functions.invoke('generate-quest', {
  body: {
    card: selectedCard,
    city: '上海·静安区',

    lng: 121.4737,
    time_period: '下午',

  }

}

console.log('生成成功:', data.journey);
```
